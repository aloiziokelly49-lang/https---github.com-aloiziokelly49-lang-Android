## 功能点 1：多格式文件上传 和 文档纵向连续滑动浏览

**支持 PDF/JPG/PNG/TXT 等多格式文件从本地沙盒上传（包含单文件与多文件并发解析）。文档展示层使用 DocumentViewerActivity，借助 Android PdfRenderer API 结合 RecyclerView，实现文档的高清纵向连续滑动浏览。**

### 1.1 多格式文件上传的设计思路

**实现思路：** 首页 `HomeFragment` 通过 Android 系统文件选择器 (`ActivityResultContracts.OpenDocument` / `OpenMultipleDocuments`) 拉起系统文件选择界面，允许用户单选或多选文件。选择后根据 MIME 类型和文件扩展名自动分类（PDF / 图片 / 文本 / 音频），并路由到不同的处理页面。对于选择的文件，通过 `ContentResolver.takePersistableUriPermission()` 获取持久化读取权限，确保后续可跨 Activity 访问。

**关键技术：**
- Android Storage Access Framework (SAF) 的 `OpenDocument` / `OpenMultipleDocuments` 契约
- `ContentResolver.getType()` 获取 MIME 类型
- `OpenableColumns.DISPLAY_NAME` 查询文件名作为备选分类依据

**关键代码文件：** `app/src/main/java/com/cloudink/app/ui/home/HomeFragment.java`

```java
// 单文件选择器
private final ActivityResultLauncher<String[]> docPicker =
    registerForActivityResult(new ActivityResultContracts.OpenDocument(), uri -> {
        if (uri == null) return;
        requireContext().getContentResolver().takePersistableUriPermission(
            uri, Intent.FLAG_GRANT_READ_URI_PERMISSION);
        routeImportedDocument(uri);
    });

// 多文件选择器
private final ActivityResultLauncher<String[]> multiDocPicker =
    registerForActivityResult(new ActivityResultContracts.OpenMultipleDocuments(), uris -> {
        if (uris == null || uris.isEmpty()) return;
        for (Uri u : uris) {
            requireContext().getContentResolver().takePersistableUriPermission(
                u, Intent.FLAG_GRANT_READ_URI_PERMISSION);
        }
        if (uris.size() == 1) {
            routeImportedDocument(uris.get(0));
        } else {
            ArrayList<String> list = new ArrayList<>();
            for (Uri u : uris) list.add(u.toString());
            Intent intent = new Intent(requireContext(), DocumentViewerActivity.class);
            intent.putStringArrayListExtra(DocumentViewerActivity.EXTRA_URI_LIST, list);
            startActivity(intent);
        }
    });

// 根据 MIME 类型 + 扩展名分类
private ImportType resolveImportType(Uri uri) {
    String type = requireContext().getContentResolver().getType(uri);
    if (type != null) {
        if (type.startsWith("audio/")) return ImportType.AUDIO;
        if ("application/pdf".equals(type)) return ImportType.PDF;
        if (type.startsWith("image/")) return ImportType.IMAGE;
        if (type.startsWith("text/")) return ImportType.TEXT;
    }
    // 降级：通过文件扩展名判断
    String name = queryDisplayName(uri);
    if (name != null) {
        String lower = name.toLowerCase();
        if (lower.endsWith(".mp3") || lower.endsWith(".m4a") || lower.endsWith(".wav")
            || lower.endsWith(".aac") || lower.endsWith(".ogg") || lower.endsWith(".flac")) {
            return ImportType.AUDIO;
        }
        if (lower.endsWith(".pdf")) return ImportType.PDF;
        if (lower.endsWith(".jpg") || lower.endsWith(".png")) return ImportType.IMAGE;
        if (lower.endsWith(".txt") || lower.endsWith(".md")) return ImportType.TEXT;
    }
    return ImportType.UNKNOWN;
}
```

**解释：** `OpenDocument` 和 `OpenMultipleDocuments` 分别处理单文件和多文件选择。多文件选择时，单文件直接路由，多文件通过 URI 列表传递到 `DocumentViewerActivity` 支持前后翻页浏览。`resolveImportType()` 优先用 MIME 类型判断文件类别，MIME 不可用时降级为扩展名判断，确保各种来源的文件都能正确分类路由。

### 1.2 PDF 高清纵向连续滑动浏览

**实现思路：** 使用 Android 原生 `PdfRenderer` API（零第三方 PDF 库依赖）逐页渲染 PDF 页面为 Bitmap，通过 `RecyclerView` + `LinearLayoutManager` 实现纵向连续滑动浏览。每页按 RecyclerView 实际宽度等比缩放渲染，保证高清显示。

**关键技术：**
- `android.graphics.pdf.PdfRenderer`：Android 5.0+ 原生 PDF 渲染 API
- `RecyclerView` + `LinearLayoutManager`：纵向连续滑动
- 内存策略：`onBindViewHolder` 时打开 PdfRenderer.Page → 渲染 → 关闭 Page，`onViewRecycled` 时回收 Bitmap，防止内存泄漏

**关键代码文件：**
- `app/src/main/java/com/cloudink/app/document/PdfDocumentLoader.java`（PDF 加载器）
- `app/src/main/java/com/cloudink/app/document/PdfViewerAdapter.java`（RecyclerView 适配器）
- `app/src/main/java/com/cloudink/app/ui/document/DocumentViewerActivity.java`（查看器主 Activity）

**PdfDocumentLoader 关键代码：**

```java
// 将 content:// URI 稳定打开为 PdfRenderer（必要时复制到缓存）
public static OpenResult open(Context context, Uri uri) throws IOException {
    // 对 file:// 直接打开
    if ("file".equalsIgnoreCase(uri.getScheme())) {
        File f = new File(uri.getPath());
        if (f.exists() && f.canRead()) {
            ParcelFileDescriptor fd = ParcelFileDescriptor.open(f, MODE_READ_ONLY);
            return new OpenResult(new PdfRenderer(fd), fd, null);
        }
    }
    // 先尝试直接打开 content://
    ParcelFileDescriptor pfd = context.getContentResolver().openFileDescriptor(uri, "r");
    if (pfd != null) {
        try {
            PdfRenderer renderer = new PdfRenderer(pfd);
            return new OpenResult(renderer, pfd, null);
        } catch (Exception e) {
            // 失败则降级：复制到缓存再打开
        }
    }
    // 降级方案：复制 URI 内容到缓存文件
    File out = new File(cacheDir, "doc_" + System.currentTimeMillis() + ".pdf");
    copyUriToFile(context, uri, out);
    ParcelFileDescriptor cacheFd = ParcelFileDescriptor.open(out, MODE_READ_ONLY);
    return new OpenResult(new PdfRenderer(cacheFd), cacheFd, out);
}
```

**解释：** `PdfDocumentLoader.open()` 采用"对 file:// 直接打开 → 对 content:// 先尝试直接打开 → 失败则复制到本地缓存"的三级策略。因为某些 content:// URI 返回的文件描述符不支持 `PdfRenderer` 直接读取（如部分网盘应用），所以设计了降级到应用缓存目录的方案，确保兼容性。

**PdfViewerAdapter 关键代码：**

```java
@Override
public void onBindViewHolder(@NonNull PageViewHolder holder, int position) {
    // 打开 PDF 页面
    PdfRenderer.Page page = pdfRenderer.openPage(position);
    // 按 RecyclerView 宽度等比计算渲染高度
    int pageW = w;
    int pageH = (int) (page.getHeight() * ((float) pageW / page.getWidth()));
    // 创建 Bitmap 并渲染
    Bitmap bmp = Bitmap.createBitmap(pageW, pageH, Bitmap.Config.ARGB_8888);
    bmp.eraseColor(0xFFFFFFFF);  // 填充白色背景
    page.render(bmp, null, null, PdfRenderer.Page.RENDER_MODE_FOR_DISPLAY);
    page.close();
    holder.imageView.setImageBitmap(bmp);
}

@Override
public void onViewRecycled(@NonNull PageViewHolder holder) {
    // 回收 Bitmap，防止内存泄漏
    if (holder.bitmap != null && !holder.bitmap.isRecycled()) {
        holder.bitmap.recycle();
    }
    holder.bitmap = null;
    holder.imageView.setImageBitmap(null);
}
```

**解释：** 这是 RecyclerView 适配器的核心渲染逻辑。每页根据 RecyclerView 实际宽度等比计算高度，通过 `PdfRenderer.Page.render()` 以 `RENDER_MODE_FOR_DISPLAY`（高清显示模式）渲染。`onViewRecycled` 中主动回收 Bitmap，防止长文档滑动时 OOM。

**DocumentViewerActivity 中多格式分发逻辑：**

```java
private void openUri(Uri uri) {
```

## 功能点 2：调用相机/相册选取纸质文稿后，进行 OCR 识别，提取文本并流转至草稿箱

**用户调用 CameraX API 拍摄或通过相册选取纸质文稿后，采用"百度云端 OCR 优先 + Tesseract 离线兜底"的双引擎策略完成文字提取。提取后的文本一键流转至手写排版引擎的底层草稿箱。**

### 2.1 设计与架构

**实现思路：** 采用"CameraX 实时预览 + 百度云端 OCR 优先 + Tesseract 离线兜底"的双引擎策略。用户拍照或选图后，先在后台线程对图片做降采样（限制最大边长 2048px，防止网络传输/本地处理过大），然后优先调用百度 OCR 手写识别 API，失败或未配置时自动回退到 Tesseract 离线 OCR。识别结果通过 `CloudInkRepository.saveDraft()` 存入 Room 草稿表，再携带 OCR 文本启动 `HandwriteEditorActivity`。

**关键技术：**
- **CameraX**（`androidx.camera`）：接管相机预览与拍照，绑定 Activity 生命周期
- **百度 OCR API**（`aip.baidubce.com`）：手写文字识别（`/rest/2.0/ocr/v1/handwriting`）+ 通用文字识别（`/rest/2.0/ocr/v1/general_basic`）
- **Tesseract OCR**（离线）：作为兜底方案，无需网络
- **图片降采样**：`BitmapFactory.Options.inSampleSize` 防止过大图片导致处理慢/OOM

**关键代码文件：**
- `app/src/main/java/com/cloudink/app/ui/ocr/CameraOcrActivity.java`（相机拍照 OCR）
- `app/src/main/java/com/cloudink/app/ocr/OcrRecognizer.java`（OCR 统一入口 + 降级策略）
- `app/src/main/java/com/cloudink/app/ocr/BaiduOcrHelper.java`（百度 OCR 调用）
- `app/src/main/java/com/cloudink/app/ui/home/HomeFragment.java`（相册选图 OCR）

### 2.3 OCR 双引擎降级策略

**关键代码：**

```java
// OcrRecognizer.java — 统一入口，百度优先 → Tesseract 离线兜底
public static void recognize(Context context, Bitmap bitmap, boolean preferHandwriting, Callback callback) {
    if (BaiduOcrConfig.isConfigured()) {
        // 百度在线 OCR（手写识别 / 通用识别）
        BaiduOcrHelper baidu = new BaiduOcrHelper();
        if (preferHandwriting) baidu.recognizeHandwriting(bitmap, baiduCb);
        else baidu.recognizeGeneral(bitmap, baiduCb);
    } else {
        // 直接走 Tesseract 离线
        runTesseractFallback(context, bitmap, preferHandwriting, callback, null);
    }
}
```

**解释：** `OcrRecognizer.recognize()` 是整个 OCR 模块的统一入口。它首先检查百度 OCR 是否已配置，已配置则优先使用（准确率更高），百度请求失败或配置缺失时自动回退到 Tesseract 本地离线识别。这种策略保证了应用在网络不佳或百度密钥未配置时仍可正常工作。

## 功能点 3：手写模拟渲染、字体选择、排版参数动态配置（字距、行距、纸张背景、笔触）

**参考 text-to-handwriting 开源项目的抖动算法思想，自研 Android Canvas 手写渲染引擎。用户在配置页内可选择多种手写字体，还可以实时拖动滑块，干预字距、行距以及高斯抖动阈值，同时支持一键切换物理纸张背景与多种物理笔触类型，预览区实时响应变化。**

### 3.1 手写渲染引擎设计

**实现思路：** 自研 `HandwritingEngine` 类，使用 Android `Canvas` + 真实 `.ttf` 手写字体逐字渲染。核心思路是：基于文本内容的哈希值生成确定性随机种子，对每个字符在 X/Y 方向施加高斯抖动偏移，模拟真实手写的自然不规则感。排版采用先分行再逐字绘制的方式，支持自动换行。

**关键技术：**
- **Android Canvas 逐字渲染**：`canvas.drawText()` 逐字符绘制，对每个 glyph 施加随机偏移
- **高斯抖动算法**：`Random(text.hashCode())` 基于文本哈希生成确定性种子，`(rnd.nextFloat() - 0.5f) * 2f * jitterScale` 生成 X/Y 偏移
- **Typeface 动态加载**：`Typeface.createFromAsset()` 从 assets/fonts/ 加载 .ttf 手写字体
- **DataBinding 双向绑定**：`HandwritingParams extends BaseObservable`，滑块修改参数后自动触发 `onPropertyChanged` 回调 → 重新渲染
- **防抖机制**：200ms 防抖延迟（`DEBOUNCE_MS = 200`），避免用户快速拖动滑块时频繁渲染

**关键代码文件：**
- `app/src/main/java/com/cloudink/app/rendering/HandwritingEngine.java`（渲染引擎）
- `app/src/main/java/com/cloudink/app/rendering/model/HandwritingParams.java`（排版参数模型）
- `app/src/main/java/com/cloudink/app/rendering/model/PenType.java`（笔触类型枚举）
- `app/src/main/java/com/cloudink/app/rendering/model/PaperThemeManager.java`（纸张主题）
- `app/src/main/java/com/cloudink/app/ui/editor/HandwriteEditorActivity.java`（编辑器界面）
- `app/src/main/java/com/cloudink/app/ui/editor/HandwriteEditorViewModel.java`（编辑器 ViewModel）

### 3.2 核心渲染算法

**关键代码：**

```java
// HandwritingEngine.java — 核心渲染入口
public Bitmap render(String text, HandwritingParams params,
                     PenType penType, int paperWidth) {
    // 1. 应用笔触属性到 Paint
    penType.applyTo(textPaint);
    textPaint.setTextSize(params.getTextSize());

    // 2. 排版分行
    List<String> lines = layoutLines(text, params, paperWidth - PADDING_H * 2);

    // 3. 计算画布高度（含基线偏移，防止第一行被裁剪）
    float lineHeight = params.getLineSpacing() * params.getTextSize();
    Paint.FontMetrics fm = textPaint.getFontMetrics();
    float baselineOffset = -fm.top;
    int totalHeight = (int)(lines.size() * lineHeight) + PADDING_V * 2 + (int)baselineOffset + PADDING_V;
    Bitmap bitmap = Bitmap.createBitmap(paperWidth, totalHeight, Bitmap.Config.ARGB_8888);
    Canvas canvas = new Canvas(bitmap);

    // 4. 绘制纸张背景（横线/方格/点阵/纯色）
    drawPaperBackground(canvas, paperWidth, totalHeight, params.getPaperIndex());

    // 5. 逐字符渲染 + 高斯抖动
    float jitter = Math.max(0f, Math.min(1f, params.getJitterThreshold()));
    Random rnd = jitter > 0.01f ? new Random(text.hashCode()) : null;  // 确定性随机
    float jitterScale = params.getTextSize() * 0.12f * jitter;

    for (String line : lines) {
        float cursorX = PADDING_H;
        for (int i = 0; i < line.length(); i++) {
            char ch = line.charAt(i);
            String glyph = String.valueOf(ch);
            float glyphW = textPaint.measureText(glyph);
            float dx = 0f, dy = 0f;
            if (rnd != null) {
                // 高斯抖动：X方向偏移 ±jitterScale，Y方向偏移 ±jitterScale*0.6
                dx = (rnd.nextFloat() - 0.5f) * 2f * jitterScale;
                dy = (rnd.nextFloat() - 0.5f) * 2f * jitterScale * 0.6f;
            }
            canvas.drawText(glyph, cursorX + dx, cursorY + dy, textPaint);
            cursorX += glyphW + charSpacingPx;
        }
        cursorY += lineHeight;
    }
    return bitmap;
}
```

**解释：** 渲染引擎的工作原理是：
1. 根据文本内容哈希生成 `Random` 实例（相同文本每次渲染结果完全一致，保证可复现性）
2. 对每个字符计算 X 和 Y 方向的随机偏移量（通过 `jitterThreshold` 参数控制抖动幅度）
3. 在 Canvas 上逐字符绘制，配合自定义字间距实现排版
4. Y 方向偏移约为 X 方向的 60%，因为真实手写中垂直偏移通常小于水平偏移

抖动强度对应的效果：
- `jitter=0`：完全规则（类似打印机）
- `jitter=0.3~0.5`：自然手写效果
- `jitter=1.0`：大幅抖动（狂野风格）

```java
// HandwritingEngine.java — 扫描可用字体
public static List<String> getAvailableFonts(Context context) {
    List<String> fonts = new ArrayList<>();
    String[] list = context.getAssets().list("fonts");
    for (String name : list) {
        if (!name.toLowerCase().endsWith(".ttf") && !name.toLowerCase().endsWith(".otf")) continue;
        // 排除拉丁/系统字体，仅展示中文手写字体
        if (name.toLowerCase().contains("barlow") || name.toLowerCase().contains("roboto")
            || name.toLowerCase().contains("arial") || name.toLowerCase().contains("sans")) {
            continue;
        }
        fonts.add("fonts/" + name);
    }
    return fonts;
}
```

## 功能点 4：录音采集与语音实时转写同步

**用户可在手写配置页直接开启语音采集。结合 Android 前台 Service 与 MediaRecorder 保证应用在息屏或切后台时，录音不中断；用户可以边录边转写（系统 SpeechRecognizer + EXTRA_PARTIAL_RESULTS）；转写的文本通过双向数据绑定 (DataBinding) 实时滚动展示在悬浮面板上，用户可随时打断并一键拼接到现有草稿中。**

### 4.1 整体架构

**实现思路：** 采用"前台 Service 录音保活 + SpeechRecognizer 实时转写 + 百度短语音识别兜底"的三层架构：

1. **AudioRecorderService**（前台 Service）：使用 `MediaRecorder` 采集 AAC 音频，通过前台通知保证息屏/切后台不中断
2. **SpeechRecognizerManager**（系统语音识别）：利用 Android 内置 `SpeechRecognizer` API，支持 `EXTRA_PARTIAL_RESULTS` 实现边录边出字
3. **BaiduAsrHelper**（百度短语音识别）：录音结束后将 PCM 数据发送百度 ASR API，作为实时转写不可用时的兜底
4. **EditorVoiceCaptureController**（控制器）：协调上述组件，通过 EventBus 接收事件，更新 `EditorVoiceState`（DataBinding 绑定的状态对象）

**关键技术：**
- **前台 Service**：`startForeground()` 保证后台录音不被系统杀死
- **MediaRecorder**：AAC 编码，44100Hz 采样率，128kbps 比特率
- **SpeechRecognizer + EXTRA_PARTIAL_RESULTS**：系统级边录边转写
- **百度短语音识别 API**（`vop.baidu.com/server_api`）：PCM 16kHz，Base64 编码上传
- **EventBus**：`RecordingStateEvent` / `AudioTranscribeEvent` 组件解耦
- **DataBinding 双向绑定**：`EditorVoiceState` 的 `liveTranscript` 字段变化自动更新 UI

```java
@Override
public void onPartialResults(Bundle partialResults) {
    ArrayList<String> matches = partialResults.getStringArrayList(
        SpeechRecognizer.RESULTS_RECOGNITION);
    if (matches != null && !matches.isEmpty()) {
        String partial = matches.get(0);
        // 通过 EventBus 发送中间结果 → UI 实时滚动展示
        EventBus.getDefault().post(new AudioTranscribeEvent(partial, false));
    }
}

// 最终识别结果
@Override
public void onResults(Bundle results) {
    String finalText = matches.get(0);
    fullTranscript.append(finalText);
    EventBus.getDefault().post(new AudioTranscribeEvent(finalText, true));
}
```

## 功能点 5：高清手写大图预览，支持双指自由缩放；长文本防 OOM 导出

**实现高清手写大图预览，支持双指自由缩放；针对长文本排版生成超大分辨率图片，容易引发内存溢出（OOM）的问题，采用分块渲染与 Canvas 拼接策略。通过 MediaStore API 将高清图片保存至系统相册，利用 PdfDocument 类将画布内容切割分页，生成标准 A4 尺寸的 PDF 文件。**

### 5.1 预览画布双指缩放

**实现思路：** 自研 `PreviewCanvas`（继承自 View），用于编辑器内预览。使用 `ScaleGestureDetector` 处理双指缩放、`GestureDetector` 处理单指拖动（Pan）。缩放范围限制在 1x~5x，通过 `Matrix` 变换实现图像的缩放和平移。

**关键技术：**
- `ScaleGestureDetector`：处理双指缩放手势
- `GestureDetector`：处理单指拖动手势
- `Matrix.postScale()` / `postTranslate()`：图像变换矩阵
- 节流重绘（`postOnAnimation`）：与 VSYNC 同步，避免 BLASTBufferQueue 溢出

**关键代码文件：**
- `app/src/main/java/com/cloudink/app/rendering/PreviewCanvas.java`（编辑器预览画布）

**PreviewCanvas 关键代码：**

```java
// PreviewCanvas.java — 双指缩放核心
private class ScaleListener extends ScaleGestureDetector.SimpleOnScaleGestureListener {
    @Override
    public boolean onScale(ScaleGestureDetector detector) {
        float newScale = scaleFactor * detector.getScaleFactor();
        newScale = Math.max(minScale, Math.min(newScale, maxScale)); // 限制 1x~5x
        float scaleChange = newScale / scaleFactor;
        // 以焦点为中心缩放
        translateX = detector.getFocusX() - scaleChange * (detector.getFocusX() - translateX);
        translateY = detector.getFocusY() - scaleChange * (detector.getFocusY() - translateY);
        scaleFactor = newScale;
        requestRedraw();
        return true;
    }
}

// 节流重绘 — 与 VSYNC 同步
private void requestRedraw() {
    if (invalidatePending) return;
    invalidatePending = true;
    postOnAnimation(() -> { invalidatePending = false; invalidate(); });
}
```

**解释：** 缩放以双指焦点为中心（`detector.getFocusX/Y`），保证用户缩放时画面不会发生跳跃。`requestRedraw()` 使用 `postOnAnimation` 节流，确保每帧最多重绘一次，与系统 VSYNC 信号同步，避免频繁 invalidate 导致的队列溢出。

### 5.2 分块渲染防 OOM 导出

**实现思路：** 长文本直接渲染到 A4 尺寸（2480×3508px，300dpi）的整张 Bitmap 会占用约 34MB 内存，多层处理极易 OOM。因此采用"分块渲染 + 拼接"策略：
1. 将文本按段落分组（每组约 15 段），每组独立渲染为一个 tile Bitmap
2. 限制单个 tile 高度 ≤ 4096px，防止单个 tile 过大
3. 将所有 tile 缩放后逐块绘制到 A4 画布上，绘制完即回收 tile
4. 最终生成一份完整的 A4 高清 Bitmap

**关键技术：**
- **分块渲染**：15 段一组，控制每组高度
- **Bitmap 及时回收**：`tile.recycle()` 拼接完立即回收
- **Canvas clipRect**：防止越界绘制
- **MediaStore API**：Android 10+ 适配，`IS_PENDING` 机制保证原子写入
- **PdfDocument**：Android 原生 PDF 生成

**关键代码文件：** `app/src/main/java/com/cloudink/app/ui/export/ExportPreviewActivity.java`

**分块渲染关键代码：**

```java
// ExportPreviewActivity.java — A4 尺寸 (300dpi)
private static final int A4_W = 2480;
private static final int A4_H = 3508;
private static final int TILE_MAX_H = 4096;  // 单 tile 最大高度

// 分块渲染
private List<Bitmap> renderTiles() {
    List<Bitmap> tiles = new ArrayList<>();
    String[] paragraphs = text.split("\n", -1);
    final int groupSize = 15;  // 每组约 15 段
    for (int i = 0; i < paragraphs.length; i += groupSize) {
        StringBuilder chunk = new StringBuilder();
        for (int j = i; j < Math.min(i + groupSize, paragraphs.length); j++) {
            if (chunk.length() > 0) chunk.append('\n');
            chunk.append(paragraphs[j]);
        }
        HandwritingParams exportParams = buildExportParams();
        Bitmap tile = engine.render(chunk.toString(), exportParams, penType, EXPORT_W);
        if (tile != null) tiles.add(tile);
    }
    return tiles;
}

// A4 拼接 — 逐块绘制，绘制完即回收 tile
private Bitmap stitchToA4(List<Bitmap> tiles) {
    Bitmap a4 = Bitmap.createBitmap(A4_W, A4_H, Bitmap.Config.ARGB_8888);
    Canvas canvas = new Canvas(a4);
    canvas.drawColor(paperColor);

    int yCursor = 0;
    for (Bitmap tile : tiles) {
        float tileScale = (float) A4_W / tile.getWidth();
        int scaledH = (int) (tile.getHeight() * tileScale);
        if (yCursor + scaledH > A4_H) scaledH = A4_H - yCursor;

        canvas.save();
        canvas.clipRect(0, yCursor, A4_W, yCursor + scaledH);  // 裁剪区域
        canvas.drawBitmap(tile, m, null);
        canvas.restore();

        yCursor += scaledH;
        tile.recycle();  // 绘制完立刻回收，释放内存
        if (yCursor >= A4_H) break;
    }
    return a4;
}
```

**解释：** 分块渲染将长文本拆分为多个小的渲染任务，每块高度控制在 4096px 以内。拼接时，每个 tile 缩放后绘制到 A4 画布的对应位置，绘制完后立即 `tile.recycle()` 回收内存，确保同一时刻只有 A4 最终 Bitmap 这一张大图在内存中。`clipRect` 防止绘制超出 A4 边界。字号按 `(1240/720)` 比例放大，保证导出图片在高分辨率下仍然清晰。

```java
    ContentValues values = new ContentValues();
    values.put(MediaStore.Images.Media.DISPLAY_NAME, "CloudInk_" + ts + ".png");
    values.put(MediaStore.Images.Media.MIME_TYPE, "image/png");
```

## 功能点 6：Room 数据库 + 文件夹 + 多重标签归档

**使用 Room 数据库存储手写记录、识别草稿与录音文件，引入"文件夹 + 多重标签 (Tags)"的双重分类归档机制。采用 UUID 作为主键，保障数据唯一性；支持按时间戳（降序）、标签类别等进行查询；数据库支持增量迁移（v1→v2→v3→v4）。**

### 6.1 数据库整体设计

**实现思路：** 基于 Room（Android Jetpack 官方 ORM 框架）设计关系型数据库。核心实体包括 `HandwriteRecord`（手写记录）、`Draft`（OCR 草稿）、`AudioRecord`（录音文件）、`Tag`（标签）、`RecordTagCrossRef`（记录-标签多对多关联表）。使用 **"文件夹 + 多标签"双重分类**：每条手写记录属于一个文件夹（`folder_name` 字段），同时可以关联多个标签（通过 `RecordTagCrossRef` 中间表实现多对多关系）。

**关键技术：**
- **Room**（`androidx.room`）：Google 官方 SQLite ORM，编译时校验 SQL
- **UUID 主键**：`UUID.randomUUID().toString()`，全局唯一，支持离线生成无冲突
- **多对多关系**：中间表 `record_tag_join(record_id, tag_id)` 实现
- **LiveData 响应式查询**：DAO 返回 `LiveData<List<T>>`，UI 层自动订阅更新
- **数据库迁移**：`Migration(1,2)` / `Migration(2,3)` / `Migration(3,4)` 支持版本迭代，通过 `ALTER TABLE ADD COLUMN`

### 6.4 多标签归档（多对多关系）

**关键代码：**

```java
// Tag.java — 标签实体
@Entity(tableName = "tags")
public class Tag {
    @PrimaryKey @NonNull private String id = UUID.randomUUID().toString();
    @ColumnInfo(name = "name")  private String name;
    @ColumnInfo(name = "color") private int color;  // 标签颜色
}

// RecordTagCrossRef.java — 多对多中间表
@Entity(tableName = "record_tag_join",
    primaryKeys = {"record_id", "tag_id"})
public class RecordTagCrossRef {
    @NonNull @ColumnInfo(name = "record_id") private String recordId;
    @NonNull @ColumnInfo(name = "tag_id")    private String tagId;
}

// HandwriteRecordDao.java — 按标签查询
@Query("SELECT DISTINCT h.* FROM handwrite_records h "
     + "INNER JOIN record_tag_join j ON h.id = j.record_id "
     + "WHERE j.tag_id = :tagId ORDER BY h.updated_at DESC")
LiveData<List<HandwriteRecord>> getRecordsByTag(String tagId);

// CloudInkRepository.java — 保存记录时自动关联标签
private void linkTags(String recordId, List<String> tagNames) {
    for (String name : tagNames) {
        Tag tag = tagDao.findByName(name.trim());
        if (tag == null) {
            tag = new Tag();
            tag.setName(name.trim());
            tag.setColor(0xFF1A73E8);
            tagDao.insert(tag);  // 标签不存在则自动创建
        }
        // 插入关联关系（避免重复）
        if (handwriteDao.countCrossRef(recordId, tag.getId()) == 0) {
            handwriteDao.insertTagCrossRef(new RecordTagCrossRef(recordId, tag.getId()));
        }
    }
}
```

**解释：** 多对多关系通过中间表 `record_tag_join` 实现。一条手写记录可以关联多个标签（如"古诗"+"演示"），一个标签也可以被多条记录使用。`getRecordsByTag()` 通过 SQL JOIN 查询特定标签下的所有记录。`linkTags()` 在保存记录时自动创建未存在的标签并建立关联。

### 6.5 数据库迁移策略

**关键代码：**

```java
// v1 → v2：为 handwrite_records 添加排版参数字段
private static final Migration MIGRATION_1_2 = new Migration(1, 2) {
    @Override
    public void migrate(@NonNull SupportSQLiteDatabase db) {
        db.execSQL("ALTER TABLE handwrite_records ADD COLUMN char_spacing REAL NOT NULL DEFAULT 0.5");
        db.execSQL("ALTER TABLE handwrite_records ADD COLUMN line_spacing REAL NOT NULL DEFAULT 1.6");
        db.execSQL("ALTER TABLE handwrite_records ADD COLUMN jitter_threshold REAL NOT NULL DEFAULT 0.35");
    }
};

// v2 → v3：添加字体路径字段
private static final Migration MIGRATION_2_3 = new Migration(2, 3) {
    @Override
    public void migrate(@NonNull SupportSQLiteDatabase db) {
        db.execSQL("ALTER TABLE handwrite_records ADD COLUMN font_path TEXT NOT NULL DEFAULT 'fonts/NiHeWoDeLangManYuZhou-2.ttf'");
    }
};

// v3 → v4：添加存储路径字段
private static final Migration MIGRATION_3_4 = new Migration(3, 4) {
    @Override
    public void migrate(@NonNull SupportSQLiteDatabase db) {
        db.execSQL("ALTER TABLE handwrite_records ADD COLUMN storage_path TEXT NOT NULL DEFAULT '内部存储/历史档案室'");
    }
};
```

**解释：** 数据库迁移通过 `Migration` 类定义，在 `Room.databaseBuilder().addMigrations()` 中注册。每次版本升级通过 `ALTER TABLE ADD COLUMN` 添加新字段，使用 DEFAULT 值兼容旧数据。设计为增量迁移链（v1→v2→v3→v4），每个迁移只需处理相邻版本之间的差异。

## 功能点 7：登录状态管理 + Jetpack DataStore 偏好存储

**启动应用时自动读取登录状态，首次登录强制引导至资料完善页面，后续登录跳过；使用 Jetpack Preferences DataStore，支持用户排版偏好的全局本地存储与无损自动加载，如滑块默认阈值、默认纸张索引等。**

### 7.1 登录状态流程

**实现思路：** 使用 Jetpack Preferences DataStore（基于 RxJava3）作为轻量级键值存储。`KEY_IS_LOGGED_IN` 标记登录状态，`KEY_IS_FIRST_LOGIN` 标记是否首次登录。应用启动时，`LoginActivity.onCreate()` 先检查登录状态：已登录则直接跳转首页，未登录则展示登录界面。首次登录成功后跳转资料完善页，后续登录跳过。

**关键技术：**
- **Jetpack Preferences DataStore + RxJava3**：`RxPreferenceDataStoreBuilder` / `RxDataStore<Preferences>`
- **ViewModel + LiveData**：MVVM 架构，登录逻辑在 ViewModel 中
- **ActivityResultLauncher**：权限请求

**关键代码文件：**
- `app/src/main/java/com/cloudink/app/data/preferences/AppPreferences.java`（偏好存储管理）
- `app/src/main/java/com/cloudink/app/ui/login/LoginActivity.java`（登录页面）
- `app/src/main/java/com/cloudink/app/ui/login/LoginViewModel.java`（登录 ViewModel）

### 7.3 DataStore 排版偏好存储

**关键代码：**

```java
// AppPreferences.java — Jetpack Preferences DataStore
public class AppPreferences {
    private static final String PREF_NAME = "cloudink_settings";

    // 登录相关键（3个）
    public static final Preferences.Key<Boolean> KEY_IS_LOGGED_IN =
        PreferencesKeys.booleanKey("is_logged_in");
    public static final Preferences.Key<String> KEY_USER_PHONE =
        PreferencesKeys.stringKey("user_phone");
    public static final Preferences.Key<Boolean> KEY_IS_FIRST_LOGIN =
        PreferencesKeys.booleanKey("is_first_login");

    // 排版偏好键（7个）
    public static final Preferences.Key<Float> KEY_DEFAULT_CHAR_SPACING =
        PreferencesKeys.floatKey("default_char_spacing");    // 字距
    public static final Preferences.Key<Float> KEY_DEFAULT_LINE_SPACING =
        PreferencesKeys.floatKey("default_line_spacing");    // 行距
    public static final Preferences.Key<Float> KEY_DEFAULT_JITTER =
        PreferencesKeys.floatKey("default_jitter");          // 抖动阈值
    public static final Preferences.Key<Integer> KEY_DEFAULT_PAPER_INDEX =
        PreferencesKeys.intKey("default_paper_index");       // 纸张索引
    public static final Preferences.Key<String> KEY_DEFAULT_PEN_TYPE =
        PreferencesKeys.stringKey("default_pen_type");       // 默认笔触
    public static final Preferences.Key<String> KEY_DEFAULT_FONT_PATH =
        PreferencesKeys.stringKey("default_font_path");      // 默认字体路径
    public static final Preferences.Key<String> KEY_DISPLAY_NAME =
        PreferencesKeys.stringKey("display_name");           // 用户显示名

    // 共 11 个 Key（登录3 + 排版6 + 字体1 + 显示名1）
    private final RxDataStore<Preferences> dataStore;

    public AppPreferences(@NonNull Context context) {
        dataStore = new RxPreferenceDataStoreBuilder(context, PREF_NAME).build();
    }

    // 读取用户偏好（带默认值降级）
    public float getDefaultLineSpacing() {
        return getBlocking(KEY_DEFAULT_LINE_SPACING, 1.5f);     // DataStore 默认 1.5f
    }

    public float getDefaultJitter() {
        return getBlocking(KEY_DEFAULT_JITTER, 0.3f);           // DataStore 默认 0.3f
    }

    // 通用阻塞读取（带默认值）
    private <T> T getBlocking(Preferences.Key<T> key, T defaultValue) {
        try {
            return (T) dataStore.data().firstOrError()
                .map(prefs -> {
                    T val = prefs.get(key);
                    return val != null ? val : defaultValue;
                })
                .subscribeOn(Schedulers.io()).blockingGet();
        } catch (Exception e) { return defaultValue; }
    }

    // 通用异步写入
    private <T> void put(Preferences.Key<T> key, T value) {
        dataStore.updateDataAsync(prefs -> {
            MutablePreferences mutable = prefs.toMutablePreferences();
            mutable.set(key, value);
            return Single.just(mutable);
        }).subscribeOn(Schedulers.io()).subscribe();
    }
}
```

**解释：** 使用 Jetpack Preferences DataStore 替代传统的 SharedPreferences，主要优势：
1. **类型安全**：`PreferencesKeys.floatKey()` / `intKey()` 等提供编译时类型检查
2. **异步 API**：读写操作在 `Schedulers.io()` 上执行，不阻塞主线程
3. **一致性保证**：通过 `updateDataAsync` 的原子更新避免并发写入问题

排版偏好的两层默认值设计：DataStore 层（lineSpacing=1.5f, jitter=0.3f）提供更保守的出厂值，`HandwritingParams` 运行时的构造函数默认值（1.6f, 0.35f）提供更自然的初始手写效果。用户修改后 DataStore 层覆盖运行时层。

### 7.4 排版偏好的自动加载

**关键代码：**

```java
// HandwriteEditorViewModel.java — 启动时加载用户偏好
private void loadPreferencesIntoParams() {
    AppPreferences prefs = CloudInkApplication.getInstance().getPreferences();
    params.setCharSpacing(prefs.getDefaultCharSpacing());    // 加载字距
    params.setLineSpacing(prefs.getDefaultLineSpacing());    // 加载行距
    params.setJitterThreshold(prefs.getDefaultJitter());     // 加载抖动
    params.setPaperIndex(prefs.getDefaultPaperIndex());      // 加载纸张
    params.setPenType(prefs.getDefaultPenType());            // 加载笔触
    String font = prefs.getDefaultFontPath();
    if (font != null && !font.isEmpty()) {
        params.setFontPath(font);
        engine.switchFont(CloudInkApplication.getInstance(), font); // 加载字体
    }
}
```

**解释：** 每次进入编辑器时，系统自动从 DataStore 读取用户上次保存的排版偏好，同步到 `HandwritingParams` 对象中（DataBinding 双向绑定自动更新 UI 滑块），同时切换引擎字体。这意味着用户的排版习惯（如"我喜欢用圆珠笔+横线纸+0.5 抖动"）会被自动记忆，每次打开编辑器都无需重新调节。
