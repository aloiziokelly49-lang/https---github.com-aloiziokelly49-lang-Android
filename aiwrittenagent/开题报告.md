
一、 开发APP的背景与目的
项目背景：
1. 随着移动终端技术的飞速发展，智能手机与平板已全面渗透至数字化学习与无纸化办公场景。用户不再满足于单一的碎片化操作，而是期望实现跨终端的“内容接力”。例如在听讲或头脑风暴时，用户需要先导入多格式资料、录制并转换讲座音频，再进行无纸化笔记与手写排版。
2. 然而，现有的移动端手写排版应用普遍存在痛点。一方面，市面上的手写模拟软件多依赖前端网页技术封装，在移动端运行卡顿，且无法与系统底层文件系统深度整合；另一方面，传统的工具软件将“资料阅读”、“音频采集”、“图片文字提取”与“排版渲染”完全剥离，用户体验极其割裂。
3. 基于上述背景，本项目拟开发一款名为 “云墨 (CloudInk)” 的现代 Android 移动应用程序。该应用以 Android 原生底层渲染技术为核心，围绕多源数据流采集与数字手账笔记主题，融合了多格式文档导入、端侧图片转文字 (OCR)、课堂录音实时转写 (ASR)、分屏双开阅读、多媒体文件导出以及多重分类的历史档案管理功能。
开发目的：
1. 深度集成 Android 原生生态与成熟开源方案： 引入 Google ML Kit 视觉引擎与 CameraX API 实现极速离线 OCR；结合 Android Service、MediaRecorder 以及SpeechRecognizer实现边录边转写；基于开源数学模型实现高性能的硬件加速手写排版渲染，规避 WebView 带来的性能瓶颈。
2. 构建完整的多模态工具闭环： 跑通从“多格式文档导入（基于 PdfRenderer API） / OCR 与语音提取 -> 分屏双开摘录（基于 EventBus 数据流转） -> 基于开源算法的高清排版渲染 -> 系统多媒体沙盒导出（结合 BitmapRegionDecoder 与 PdfDocument）”的全链路业务流。
3. 综合运用现代 Android 架构： 深度运用 Jetpack 组件（如 Room 数据库、Preferences DataStore），结合双向数据绑定（DataBinding）处理 UI 状态的实时更新，展现高标准的软件工程规范。
4. 打造高分课程项目： 构建一个结构严谨、功能边界清晰、技术亮点突出的综合性项目，满足期末现场演示、功能压测与答辩讲解的严苛要求。
二、 主要页面简介
1. 登录与资料页 (LoginActivity & ProfileActivity)： 应用身份验证入口。支持手机号与密码本地鉴权，首次登录引导用户完善排版偏好（如滑块默认阈值、默认纸张索引），基于 Jetpack Preferences DataStore 保障配置文件的本地隔离、持久化存储与无损加载。
2. 首页功能台 (HomeFragment)： 应用主功能调度枢纽。采用 Material Design 3 的模块化网格卡片布局，集中提供“手写模拟器”、“文档导入”、“相册/相机 OCR”、“语音转写库”等核心入口。
3. 分屏文档阅读与摘录页 (SplitReaderActivity)： 采用左右双栏分屏交互设计。左侧借助 Android PdfRenderer API 结合 RecyclerView 实现文档视图（SheetViewerScreen），右侧为草稿面板。通过 EventBus 支持选中文本一键提取至右侧，打造高效的文献处理工作台。
4. 手写渲染引擎配置页 (HandwriteEditorActivity)： 本项目最核心的功能页面。上半部分为“所见即所得”的实时手写预览区；下半部分为滑动参数控制台。内嵌悬浮式语音采集面板，利用 DataBinding 双向绑定技术，支持边录音边将转化文本流式实时滚动并追加至排版区。
5. 高清大图预览与导出页 (ExportPreviewActivity)： 用于展示最终渲染排版成果。采用 BitmapRegionDecoder 技术支持双指缩放查看超大尺寸高清长图，有效防范内存溢出。底部提供基于 MediaStore API 的“保存至相册”与基于 PdfDocument 的“导出为 PDF”选项。
6. 历史档案室与标签管理页 (HistoryFragment)： 应用的本地数据管理中心。采用 Room 数据库按时间线展示历史手写记录、识别草稿与录音文件；带有独立音频回放控件，并提供“文件夹 + 自定义标签 (Tags)”的双重管理体系。
7. 个人中心与设置页 (ProfileFragment)： 包含基础状态信息、缓存容量清理，以及手写模块的全局默认偏好持久化设置中心。
三、 主要页面设计
(说明：在最终文档中需插入实际 UI 原型或使用手机/模拟器生成的截图)
1. 首页功能台设计： 顶部采用非对称渐变色卡片展示用户信息。中部功能矩阵采用圆角阴影卡片 (CardView) 搭配矢量图标，将“文字提取”与“手写模拟”高亮呈现，符合现代效率工具的极简美学。
2. 手写引擎配置页设计： 视觉重心居中。上方为自定义预览画布 (PreviewCanvas)，模拟物理纸张纹理；下方控制台通过 BottomSheet 呈现，内置多个 Slider 滑块。侧边设有一键展开的悬浮胶囊录音组件，展开后显示音频波形动画与动态滚动的转写文本。
3. 分屏文档阅读页设计： 页面中心设有可左右拖拽的动态分割线（基于 ConstraintLayout 和触摸事件分发拦截）。左侧长按文字提供高亮反馈；右侧摘录板以便利贴堆叠形式呈现，交互逻辑顺畅自然。
4. 历史档案室设计： 采用瀑布流或列表卡片布局 (RecyclerView)。顶部展示文字摘要及背景纸张缩略图，底部横向排布彩色标签（ChipGroup）。含录音的笔记卡片会自动内嵌极简的音频播放控制条。
四、 用户界面的内部功能介绍
本应用的核心技术难点集中在移动端的多媒体处理、本地性能优化与复杂界面交互，具体实现亮点如下：
功能点 1：多格式文件上传 和 文档纵向连续滑动浏览
支持 PDF/JPG/PNG/TXT 等多格式文件从本地沙盒上传（包含单文件与多文件并发解析）。文档展示层使用 SheetViewerScreen，借助 Android PdfRenderer API 结合 RecyclerView，实现文档的高清纵向连续滑动浏览。
功能点 2：调用相机/相册选取纸质文稿后，进行 OCR 识别，提取文本并流转至草稿箱
集成 Google ML Kit Text Recognition 框架，用户调用 CameraX API 拍摄或通过相册选取纸质文稿后，系统能离线提取纯文本。支持简单的边界裁剪与透视校正，提取后的文本一键流转至手写排版引擎的底层草稿箱。
功能点 3：手写模拟渲染 与 排版参数动态配置（字距、行距、纸张背景、笔触）
基于 文字转手写开源数学模型（如saurabhdaware/text-to-handwriting），实现手写模拟渲染 和 排版配置。 用户在配置页内可实时拖动滑块，干预字距、行距以及高斯抖动阈值，同时支持一键切换物理纸张背景与多种物理笔触类型，预览区实时响应变化。
功能点 4：双栏分屏阅读与文献划线协同摘录
通过事件总线 EventBus，支持在左右双栏水平分屏模式下进行文献阅读，划线摘录后内容可自动追加至右侧草稿箱。
功能点 5：录音采集与语音实时转写同步
用户可在手写配置页直接开启语音采集。结合 Android Service 与 MediaRecorder 保证应用在息屏或切后台时，录音不中断。利用系统内置 SpeechRecognizer 实现边录边转写。转写的文本通过双向数据绑定 (DataBinding) 实时滚动展示在悬浮面板上，用户可随时打断并一键拼接到现有草稿中。
功能点 6：实现高清手写大图预览，支持双指自由缩放；针对长文本排版生成超大分辨率图片，容易引发 内存溢出 的问题，采用分块绘制与 BitmapRegionDecoder 技术。通过 MediaStore API 将 高清图片保存至系统相册，利用 PdfDocument 类将画布内容切割分页，生成标准 A4 尺寸的 PDF 文件。
功能点 7：使用 Room 数据库 存储 手写记录、识别草稿与录音文件，引入“文件夹+多重标签 (Tags)”的双重分类归档机制。采用 UUID 作为主键，保障数据唯一性；支持按时间戳（降序）、标签类别等进行查询。
功能点 8： 启动应用时自动读取登录状态，首次登录强制引导至资料完善页面，后续登录跳过；使用Jetpack Preferences DataStore ，支持用户排版偏好的全局本地存储与无损自动加载，如 滑块默认阈值、默认纸张索引等
