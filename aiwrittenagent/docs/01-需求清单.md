# 云墨 (CloudInk) — 需求清单（STAR 法则）

> **项目名称**: 云墨 CloudInk | **平台**: Android (Java) | **日期**: 2026-05-23

---

## S — Situation（背景与现状）

| 编号 | 背景描述 | 痛点/机会 |
|------|----------|-----------|
| S1 | 移动端手写排版应用多基于 WebView 封装 | 渲染卡顿、无法与系统文件系统深度整合 |
| S2 | 资料阅读 + 音频采集 + OCR + 排版渲染工具割裂 | 用户需要多个 App 切换，体验割裂 |
| S3 | 学生/办公群体需要"内容接力"（听课录音→转写→笔记→导出） | 缺少一站式多模态闭环工具 |
| S4 | 现有 OCR/ASR 方案依赖云端，离线不可用 | 需要端侧离线识别能力 |
| S5 | 长文本手写排版渲染易导致内存溢出 | 需要分块绘制与高效解码技术 |

## A — Action（非功能性需求 / 技术约束）

| 编号 | 类别 | 约束/要求 |
|------|------|-----------|
| A1 | 架构 | 采用 Jetpack 组件体系（Room, DataStore, ViewModel, LiveData） |
| A2 | 数据绑定 | UI 状态更新统一使用 DataBinding 双向绑定 |
| A3 | 渲染 | 禁止使用 WebView 做手写渲染，必须使用 Android 原生 Canvas |
| A4 | OCR | 使用 Google ML Kit 实现离线 OCR，不依赖云端 API |
| A5 | ASR | 使用系统 SpeechRecognizer，录音使用 MediaRecorder + Service |
| A6 | 文档 | PDF 渲染使用 PdfRenderer API，禁止第三方 PDF 库 |
| A7 | 导出 | 图片存入系统相册使用 MediaStore API（适配 Android 10+ 分区存储） |
| A8 | 通信 | 组件间解耦通信使用 EventBus |
| A9 | 性能 | 大图加载必须使用 BitmapRegionDecoder 分块，防止 OOM |
| A10 | 数据库 | Room + UUID 主键，支持迁移 |
| A11 | 开源合规 | 所有引入的开源库必须记录许可证类型，确保与项目兼容 |
| A12 | 最低版本 | Android API Level 26 (Android 8.0) |

## R — Result（验收标准）

| 编号 | 验收项 | 通过标准 |
|------|--------|----------|
| R1 | 登录流程 | 手机号+密码登录成功 → 首次用户跳转资料页 → 二次用户直接进首页 |
| R2 | OCR 识别 | 拍照/选图后 3 秒内返回可编辑文本，离线可用 |
| R3 | 手写渲染 | 滑块调节 200ms 内实时刷新预览，纸张切换无闪烁 |
| R4 | 分屏摘录 | 左侧选中文本 → 右侧实时追加，无 EventBus 粘性事件残留 |
| R5 | 录音转写 | 息屏录音不中断，转写文本实时滚动追加 |
| R6 | 大图导出 | 10000px+ 长图预览无 OOM，导出图片/PDF 可在系统相册/文件管理器打开 |
| R7 | 历史管理 | 支持按标签/时间查询，1000 条记录列表滑动不卡顿 |
| R8 | 偏好持久化 | 杀死进程重启后偏好不丢失 |
| R9 | 开源合规 | 可输出完整的依赖许可证清单 |
