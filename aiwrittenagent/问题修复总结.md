# 问题修复总结

## 📌 修复的三个主要问题

### 1. ✅ PDF文件导入后左侧不显示

**原因分析：**
- RecyclerView初始化时宽度为0
- PDF渲染时机过早，布局还未完成
- 缺少错误日志，难以定位问题

**解决方案：**
- 添加了详细的日志输出（可通过Logcat查看）
- 改进了宽度获取逻辑，增加多重fallback机制
- 优化了渲染时机，确保布局完成后再渲染
- 改进了布局文件，添加最小高度和白色背景

**测试方法：**
1. 打开应用 → 分屏阅读
2. 点击顶部工具栏
3. 选择PDF文件
4. 左侧应该显示PDF内容

**如果还是空白，请查看日志：**
```bash
adb logcat | grep PdfViewerAdapter
```

---

### 2. ✅ 录音文件只能播放，不能转文字

**解决方案：**
- 集成了百度语音识别API
- 添加了"识别为文字"按钮
- 支持WAV、M4A、AMR等多种音频格式

**使用步骤：**
1. 语音转写界面 → 点击"导入录音"
2. 选择录音文件
3. 点击"识别为文字"按钮
4. 等待识别完成（需要网络）
5. 文字自动添加到编辑框

**⚠️ 重要：首次使用需要配置API密钥**

**配置步骤：**
1. 访问百度AI平台：https://ai.baidu.com/
2. 注册账号 → 创建应用 → 选择"语音识别"
3. 获取API Key和Secret Key
4. 打开文件：`app/src/main/java/com/cloudink/app/asr/BaiduAsrHelper.java`
5. 找到第18-19行，替换密钥：
   ```java
   private static final String API_KEY = "你的API_KEY";
   private static final String SECRET_KEY = "你的SECRET_KEY";
   ```
6. 重新编译安装

**免费额度：** 每天50000次调用，足够个人使用

**如果不配置：** 点击识别时会显示配置提示

---

### 3. ✅ vivo设备只能用输入法语音输入

**原因：**
vivo X90等设备不支持Android系统的SpeechRecognizer API

**当前解决方案：**
使用搜狗输入法的语音输入功能

**使用方法：**
1. 语音转写界面 → 点击"打开输入法"
2. 点击搜狗键盘上的麦克风图标🎤
3. 说话后自动转文字
4. 点击发送添加到文本框

**提示：** 这个方法识别准确率很高，推荐使用

**未来改进：**
可以集成讯飞或百度的语音SDK，实现微信式按住说话功能

int w = holder.itemView.getWidth();

if (w <= 0) w = viewWidth;

## 🐛 故障排查

### PDF不显示？

**方法1：查看日志**
```bash
adb logcat | grep "PdfViewerAdapter"
```

查找关键信息：
- "PDF 加载成功" - PDF已打开
- "RecyclerView 宽度: XXX" - 宽度是否为0
- "渲染完成" - 是否成功渲染

**方法2：尝试其他PDF**
- 换一个小的PDF文件试试
- 确认PDF文件没有损坏

**方法3：重启应用**
- 完全关闭应用
- 清除应用数据
- 重新打开

### 录音识别失败？

**错误1："请先配置API密钥"**
→ 按照上面的步骤配置百度API密钥

**错误2："获取access token失败"**
→ 检查API Key和Secret Key是否正确

**错误3："识别失败: err_no=3301"**
→ 音频格式不支持，尝试转换为WAV

**错误4："Network error"**
→ 检查网络连接

### 输入法语音不工作？

**检查清单：**
- ✅ 是否安装了搜狗输入法？
- ✅ 是否设为默认输入法？
- ✅ 是否授予了麦克风权限？
- ✅ 搜狗输入法是否启用了语音功能？
