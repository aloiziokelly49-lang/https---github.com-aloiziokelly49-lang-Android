### 功能点 3：手写渲染引擎（最核心的功能点，重点准备）

**步骤 3A：看渲染入口（最重要！）**

- **打开文件：** `rendering/HandwritingEngine.java`
- **定位到：** `render()` 方法（约第 131 行）
- **看什么——按代码顺序逐行理解：**
  1. `penType.applyTo(textPaint)` — 画笔触属性（钢笔/圆珠笔/马克笔）
  2. `textPaint.setTextSize(params.getTextSize())` — 设字号
  3. `layoutLines(text, params, availableWidth)` — 先分行（按可用宽度 + 字号算每行能放几个字）
  4. `drawPaperBackground()` — 画纸张底色+纹理（横线/方格/点阵/纯色）
  5. **核心抖动逻辑：**
     ```java
     Random rnd = new Random(text.hashCode());  // 文本哈希 = 确定性种子
     dx = (rnd.nextFloat() - 0.5f) * 2f * jitterScale;      // X 偏移
     dy = (rnd.nextFloat() - 0.5f) * 2f * jitterScale * 0.6f; // Y 偏移（比 X 小 40%）
     canvas.drawText(glyph, cursorX + dx, cursorY + dy, textPaint);  // 加偏移绘制
     ```
- **记忆口诀：**
  - 渲染流程："笔触→字号→分行→背景→逐字抖动→输出 Bitmap"
  - 抖动原理："文本哈希当种子，每个字符随机抖，X 大 Y 小模拟人手"

**步骤 3B：看参数模型 + DataBinding 双向绑定**

- **打开文件：** `rendering/model/HandwritingParams.java`
- **定位到：** 类声明 `extends BaseObservable` 和每个 getter 的 `@Bindable` 注解
- **看什么：**
  - 6 个排版参数：textSize / charSpacing / lineSpacing / jitterThreshold / paperIndex / penType / fontPath
  - 每个 setter 都调 `notifyPropertyChanged(BR.xxx)` 通知 DataBinding
- **记忆口诀：** "BaseObservable + @Bindable → setter 调 notify → 自动触发重渲染"

**步骤 3C：看防抖机制**

- **打开文件：** `ui/editor/HandwriteEditorViewModel.java`
- **定位到：** `scheduleRender()` 方法（约第 168 行）和 `RenderTask` 内部类（约第 173 行）
- **看什么：**
  - `DEBOUNCE_MS = 200` — 200ms 防抖延迟
  - `debounceHandler.removeCallbacks(renderTask)` — 取消上次调度
  - `debounceHandler.postDelayed(renderTask, DEBOUNCE_MS)` — 延迟执行
  - `RenderTask.run()` 在 `renderExecutor`（单线程线程池）中执行渲染
- **记忆口诀：** "参数变→取消旧任务→等 200ms 没再变→后台线程渲染→postValue 更新 UI"

**步骤 3D：看笔触和纸张**

- **打开文件：** `rendering/model/PenType.java` 和 `rendering/model/PaperThemeManager.java`
- **看什么——笔触：**
  - FOUNTAIN（钢笔）：细(3.0px) + 深蓝黑(#1A237E) + 高透明度(220)
  - BALLPOINT（圆珠笔）：中(3.8px) + 纯黑(#1A1A1A) + 满透明度(240)
  - MARKER（马克笔）：粗(5.5px) + 红色(#D32F2F) + 半透明(160) + ShadowLayer 洇染
- **看什么——纸张：**
  - 四主题：护眼黄/极简白/深邃黑/简约灰
  - 四纹理：纯色(0)/横线(1)/方格(2)/点阵(3)
- **记忆口诀：** "笔触三要素：宽/透/色 → 马克笔加阴影洇染 | 纸张四色四纹 16 组合"

**步骤 3E：看字体动态加载**

- **打开文件：** `ui/editor/HandwriteEditorActivity.java`
- **定位到：** `setupFontToggle()` 方法（约第 248 行）
- **看什么：**

### 功能点 5：预览缩放 + 防 OOM 导出

**步骤 5A：看双指缩放**

- **打开文件：** `rendering/PreviewCanvas.java`
- **定位到：** `ScaleListener` 内部类（约第 138 行）和 `requestRedraw()` 方法（约第 87 行）
- **看什么：**
  - `detector.getFocusX/Y` — 以双指焦点为中心缩放
  - 缩放范围 `1x~5x`
  - `postOnAnimation` 节流重绘，与 VSYNC 同步
- **记忆口诀：** "ScaleGestureDetector + 焦点为中心 → postOnAnimation 节流防溢出"

**步骤 5B：看分块渲染（这是防 OOM 的核心！）**

- **打开文件：** `ui/export/ExportPreviewActivity.java`
- **定位到：** `renderTiles()` 方法（约第 165 行）和 `stitchToA4()` 方法（约第 206 行）
- **看什么——关键数字：**
  - A4 = 2480×3508px (300dpi)，单张约 34MB
  - `groupSize = 15` — 每组 15 段
  - `TILE_MAX_H = 4096` — 单 tile 不超过 4096px 高
  - `EXPORT_W = 1240` — tile 渲染宽度
- **看什么——分块流程：**
  1. 文本按 `\n` 分段 → 15 段一组 → 每组渲染为一个 tile
  2. tile 逐个缩放到 A4 宽度 → 绘制到 A4 画布 → **绘制完立即 `tile.recycle()`**
  3. `canvas.clipRect()` 防止越界
- **记忆口诀：**
  - "A4 34MB 太大 → 15段一组分块渲染 → 逐块贴上去立马回收 → 内存只保留最终一张"
  - "2480×3508 A4 → 1240 宽 tile → 15段一组 → recycle 是关键"

**步骤 5C：看保存和 PDF 导出**

- **打开文件：** `ui/export/ExportPreviewActivity.java`
- **定位到：** `saveToGallery()` 方法（约第 245 行）和 `exportPdf()` 方法（约第 291 行）
- **看什么：**
  - MediaStore 三步：insert(IS_PENDING=1) → write → update(IS_PENDING=0)

- **记忆口诀：** "IS_PENDING 原子写入 → PdfDocument 原生 A4 导出"

### 模拟问题清单

**功能点 1：**
- "你们是怎么实现多格式文件上传的？" → SAF + MIME 分类
- "PDF 是如何做到纵向连续滑动浏览的？" → PdfRenderer + RecyclerView

**功能点 3：**
